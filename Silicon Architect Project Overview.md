# **Project Overview: Silicon Architect**

**Date:** April 23, 2026  
**Project Goal:** To develop and publish a high-quality Android mobile game that blends "Merge" mechanics with "Idle Builder" strategy, focusing on high-speed development and research into tactile feedback and system optimization.

## ---

**1\. The Core Concept**

Silicon Architect is a "Motherboard Builder." Instead of generic fantasy or cafe themes, the player acts as a hardware architect. The gameplay takes place on a fixed-size PCB (Printed Circuit Board) grid where players merge components to evolve technology while managing the physical constraints of a real computer system.

### **The "Spark" (The Competitive Edge)**

Unlike standard idle clones that rely on artificial "time-gates" or premium currency to slow progress, Silicon Architect uses **System Constraints**:

* **Heat Management:** Components generate heat that bleeds into adjacent tiles. Overheating causes "Throttling," reducing efficiency.  
* **Power Draw:** Every component requires Wattage. Players must balance their "Power Rail" output against their high-tier processors.  
* **Spatial Strategy:** It is a puzzle of layout. Placing a Fan next to a CPU is a tactical choice, not just a numerical upgrade.

## ---

**2\. Gameplay Mechanics**

| Feature | Description   |
| :---- | :---- |
| **The Merge Loop** | Combine low-tier components (Transistors) into higher-tier parts (Logic Gates \> CPUs \> SOCs). |
| **Socketing** | Active components must be placed in specific "Sockets" on the motherboard to generate "Data" (the primary currency). |
| **Adjacency Bonuses** | Special components like Heatsinks or RAM provide buffs to the tiles immediately surrounding them. |
| **Manual Override** | A "Quality" feature allowing players to tap to manually cool the system, providing an active way to bypass bottlenecks. |

## ---

**3\. Technical Strategy & "Vibecoding"**

The development follows a "Speed \+ Quality" protocol using modern AI-assisted workflows.

* **Engine:** Godot 4.x (.NET Edition).  
* **Language:** C\# (Chosen for strict typing, scalability, and high Copilot accuracy).  
* **IDE:** VS Code with GitHub Copilot as a "Junior Developer" partner.  
* **Development Methodology:** System-First. The thermal and power math is built as a simulation before the visual layer is finalized.

## ---

**4\. Visual & Aesthetic Direction**

To maintain a high "Quality" feel without massive asset costs, the game uses a **Technical/Industrial Aesthetic**:

* Dark green or charcoal PCB backgrounds.  
* Glowing neon "Trace" lines that pulse based on data throughput.  
* A thermal overlay (Heatmap) that visually represents system stress through shaders.  
* Biometric-style "Boot Sequence" for app entry to enhance the "Secure Tech" atmosphere.

## ---

**5\. Roadmap for MVP (Minimum Viable Product)**

1. **Phase 1: The Grid:** Establish a 5x5 motherboard layout with basic C\# data structures.  
2. **Phase 2: Thermal Engine:** Implement the cellular automata heat-spread and power consumption logic.  
3. **Phase 3: The Loop:** Add drag-and-drop merging and the "Data" generation economy.  
4. **Phase 4: Android Polish:** Optimize touch controls and prepare for Play Store deployment.

*Note: This project serves as a dual-purpose endeavor: a market-ready productivity/game hybrid and a research case study into solo-dev speed vs. software quality.*
